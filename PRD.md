# Agent 智能体平台系统 需求说明书

## 文档说明
### 文档目的
本文档为 Agent 智能体平台的完整设计需求规范，用于指导工程化开发，输出可落地的项目工程文件。文档覆盖平台整体架构、核心组件设计、全链路流程、质量保障、成本管控等全部核心模块，所有设计要求均为明确可实现的技术规则。
### 适用范围
平台后端全量核心服务、Agent 运行时、管控后台、基础中间件集成。
### 设计总原则
1. 管控与执行分离：平台层负责调度、管控、校验，Agent 运行时仅负责单任务执行
2. 全链路可观测：所有环节留痕可追溯，支持问题排查与效果度量
3. 能力资产化：Agent、工具、知识、模板统一沉淀复用
4. 分级容错：多层级兜底保障，平衡成功率与成本
5. 安全优先：权限、合规、风控贯穿全链路

---

## 一、整体架构拆分与模块职责
采用「四层业务平面 + 两大横向体系」的分层架构，各模块高内聚低耦合，可独立迭代。

### （一）四层业务平面
#### 1. 基础资源层
为全平台提供底层标准化资源能力，屏蔽异构差异。
- **算力资源池**：容器化弹性算力，支持 Serverless 模式 Agent 实例按需拉起销毁，实现资源隔离与按量计费
- **模型服务网关**：统一接入国内外主流大模型与私有化开源模型，标准化调用协议；具备智能路由、计量统计、密钥配额管理、Prompt 缓存能力
- **存储中间件集群**：关系库（存储结构化配置、会话、任务数据）、向量库（存储记忆与知识向量）、分布式缓存（存放短期记忆与热点数据）、对象存储（存放附件与归档数据）
- **基础中间件**：消息队列、分布式锁、全文搜索引擎等通用组件

#### 2. 核心引擎层
平台核心能力中枢，负责任务全生命周期管控与 Agent 执行驱动。
- **任务编排与调度引擎**：任务状态机管理、DAG 解析与分发、依赖关系校验、异常处理、多 Agent 协同模式调度
- **智能规划引擎**：任务复杂度识别、子任务拆解、DAG 生成、动态重规划、规划策略沉淀优化
- **记忆管理引擎**：三级记忆生命周期管理、记忆加工与向量化、多路召回与重排序、记忆膨胀治理
- **统一工具引擎**：工具标准化注册、智能召回、调用网关代理、沙箱隔离执行、结果标准化清洗
- **Agent 运行时引擎**：Agent 实例生命周期管理、上下文注入、ReAct/Reflexion 推理循环驱动、Token 水位管控、执行状态同步

#### 3. 能力服务层
基于核心引擎封装的可复用业务资产层，实现能力产品化。
- **Agent 仓库**：Agent 全生命周期管理、能力标签体系、效果动态评分、低代码配置台
- **知识库服务**：多格式文档解析切片、知识版本与权限管理、知识质量校验
- **任务模板中心**：高频场景标准化任务模板、参数化配置、模板效果迭代优化
- **质量评估服务**：自动质量校验、效果评测、Badcase 归集、效果数据回流优化

#### 4. 接入交互层
面向终端用户与外部系统的统一入口。
- **统一接入网关**：支持多协议接入、身份鉴权、流量控制、请求标准化转换
- **会话管理服务**：会话全生命周期管理、多轮对话状态维护、流式输出支持、历史消息回溯
- **前端交互控制台**：运营后台、Agent 配置工作台、调试沙箱、终端对话界面
- **输出适配模块**：多渠道输出格式转换，支持富文本、文件、结构化数据等多形态输出

### （二）两大横向体系
1. **安全风控与合规体系**：覆盖内容安全、访问控制、数据合规、成本管控，贯穿全栈所有层级
2. **全链路可观测与运营体系**：包含链路追踪、指标监控、日志中心、多维度运营分析

### （三）完整执行全链路（从用户输入到任务完成）
1. **请求接入与上下文初始化**：多端接入协议标准化、前置风控与权限校验、会话历史上下文加载
2. **意图理解与任务形式化**：语义解析与实体抽取、按复杂度路由分发、生成结构化 Task Schema
3. **任务规划与多 Agent 编排**：任务复杂度评估、子任务拆解与 DAG 生成、智能匹配 Agent、定义协同策略
4. **Agent 运行与工具调用执行**：Agent 实例化与上下文注入、ReAct 推理循环、统一工具网关调用、中间状态同步落盘
5. **结果聚合与质量校验**：子任务结果归集、结果整合润色、多维度质量校验、异常兜底处置
6. **响应输出与会话收尾**：输出格式适配、结果返回、会话状态归档
7. **记忆沉淀与能力迭代**：分层记忆存储、效果自动评估、平台能力资产化迭代

## 二、核心组件详细设计要求
### （一）记忆系统设计
#### 1. 三级记忆架构
- **短期记忆（工作记忆）**：直接位于大模型上下文窗口内，存储系统提示词、当前任务目标、最近 N 轮对话、本轮工具调用历史、召回的长期记忆片段；介质为内存/分布式缓存，受 Token 窗口严格限制
- **长期记忆**：跨会话持久化存储，分为三类：
  - 情景记忆：用户交互历史、个性化偏好；采用向量库+关系库双存储
  - 语义记忆：领域知识、事实数据、业务规则；采用向量库+结构化库存储
  - 流程记忆：任务模板、最佳实践、失败解决方案；存储于模板库+向量库
  写入机制：任务完成归档、用户明确标注等触发，经信息提取、标签分类、重要性评分、向量化、去重校验后入库
  召回机制：多路召回（语义向量+关键词+时间权重+标签匹配）→ 综合重排序 → Top-N 注入上下文
- **瞬时记忆**：单轮推理内的中间思考、草稿数据，轮次结束自动丢弃，不进入对话历史

#### 2. 记忆膨胀解决方案
- **分级存储与生命周期淘汰**：按重要性评分将记忆分为热/温/冷三级，设置 TTL 自动淘汰，冷记忆归档至低成本存储
- **记忆蒸馏与压缩**：定期对同主题碎片化记忆进行蒸馏浓缩，构建「全局摘要-主题摘要-细节记忆」三级结构；高频流程类记忆转化为标准化模板
- **语义去重与关联治理**：写入时做语义相似度校验，高相似记忆执行更新而非新增；构建记忆关联图谱，按业务域分域隔离
- **召回侧精准限流**：严格控制单次召回数量，设置相关性阈值，根据上下文剩余 Token 动态调整召回配额

#### 3. 上下文 Token 阈值与压缩规则
- **四级水位线**（以 128K 上下文窗口为例）：
  - 安全水位：≤70%，正常运行无压缩
  - 预警水位：70%~85%，触发轻度压缩
  - 临界水位：85%~95%，触发中度压缩
  - 熔断水位：≥95%，触发重度压缩兜底
- **分级压缩规则**：
  - 轻度压缩：裁剪工具返回冗余字段、精简思考过程、折叠闲聊类内容
  - 中度压缩：早期对话按主题摘要化、裁剪记忆召回数量、精简系统提示词、归档已完成子任务详情
  - 重度压缩：滑动窗口保留最近 K 轮对话、工具历史仅保留结论、清空非核心补充信息
- 执行原则：所有压缩操作全量留痕，支持回溯；按任务类型动态分配配额，优先保障核心推理信息

### （二）工具能力与调用管控系统
#### 1. 工具标准化封装
所有工具遵循统一 Tool Schema 规范，分为三层：
- 元数据层：工具ID、名称、功能描述、场景标签、调用成本、平均耗时、风险等级
- 接口层：输入参数 Schema、输出结构定义、错误码规范
- 实现层：原子工具、复合工具、Agent 型工具三级封装

#### 2. 工具调用全链路标准流程
工具语义召回 → 模型生成调用指令 → 统一网关接入 → 前置校验 → 路由分发 → 隔离执行 → 结果标准化清洗 → 返回 Agent 上下文 → 全量落盘审计

#### 3. 权限边界管控
- **工具风险三级分级**：
  - R1 低危：只读公开数据、无副作用；通用执行器运行；全量 Agent 默认开放
  - R2 中危：只读内部数据、轻度可回滚副作用；内网代理执行器；按业务线申请绑定指定 Agent
  - R3 高危：写操作、系统级操作、不可回滚副作用；隔离沙箱运行；单独审批、限时授权、双人复核
- **细粒度授权模型**：采用 RBAC + ABAC 结合，遵循最小权限原则；支持临时权限自动过期、多租户数据隔离
- **核心管控机制**：高危参数白名单校验、写操作二次确认、全量调用审计留痕、越权实时拦截告警

#### 4. Token 与成本管控
- Prompt 端降本：基于语义按需召回工具注入上下文，工具 Schema 最小化，开启模型 Prompt 缓存
- 返回端降本：标准化裁剪冗余元数据，设置单工具返回 Token 上限，超长自动生成摘要，支持分页增量返回
- 调用次数管控：单任务设置最大工具调用次数熔断，相同入参调用结果缓存复用，同功能工具优先调用低成本选项，失败重试限次
- 全局成本熔断：按工具、业务线、单任务分别设置成本阈值，分级预警与自动拦截

#### 5. 业务效果量化指标
- 质量类：调用成功率、参数错误率、工具选错率、重试成功率
- 价值类：任务完成率提升幅度、结果准确率提升幅度、人工介入率下降幅度
- 成本类：单次调用平均成本、工具相关 Token 占比、缓存命中率
- 合规类：越权调用拦截率、高危工具审批合规率、审计覆盖率
- 性能类：平均调用耗时、P95 调用耗时、工具可用率、峰值并发承载能力

### （三）任务规划与编排引擎
#### 1. 任务复杂度识别
- **三级分级标准**：
  - L1 简单任务：单目标单步、无需工具与外部知识，单 Agent 直出；匹配轻量模型
  - L2 中等任务：单领域、1~3 次工具调用，单 Agent 可闭环；匹配中等模型
  - L3 复杂任务：多目标跨领域、多步骤依赖，需多 Agent 协同；全流程 DAG 编排，强模型做规划
- **识别维度**：从目标、执行、领域、知识、风险、上下文六个维度综合打分
- **两级识别流程**：规则初筛（长度、关键词、历史匹配、场景标签）→ 轻量模型精判；执行中支持动态升级复杂度

#### 2. 复杂任务规划生成逻辑
- **两种规划模式**：
  - 模板化规划：高频标准化场景复用预置 DAG 模板，仅做参数填充，效率高、结果稳定
  - 通用智能规划：长尾个性化场景调用规划大模型动态生成执行 DAG，灵活度高
- **完整规划生成流程**：
  1. 任务语义形式化：提取总目标、交付要求、约束条件、资源边界，输出标准 Task Schema
  2. 拆解策略决策：优先匹配模板，无匹配则进入智能拆解
  3. 子任务生成与能力标注：生成原子性子任务，附带输入输出、能力标签、执行配置
  4. 依赖梳理与 DAG 构建：区分数据依赖、逻辑依赖、无依赖，生成标准有向无环图，划分并行批次
  5. 规划自检优化：从完备性、原子性、效率、成本、容错五个维度校验修正
  6. 输出标准化执行计划
- **动态重规划机制**：
  触发条件：子任务连续失败无降级方案、校验不通过无法修复、核心依赖失效、用户需求变更
  模式：增量重规划（保留已完成节点，仅调整后续路径，优先采用）、全量重规划（极端场景使用，限次管控）
  约束：单任务设置重规划次数上限与成本上限，超限转入人工介入

### （四）多维度检索与代码解析系统
#### 1. 代码预处理结构化加工
- **AST 语法解析**：基于 Tree-sitter 实现多语言统一解析，精准识别函数、类、接口等代码单元，提取元数据与依赖关系，支持增量解析更新
- **代码知识图谱构建**：实体包含项目、模块、文件、类、函数、第三方依赖；关联边包含结构关系、调用关系、依赖关系；图数据库存储
- **多粒度索引并行构建**：同步构建向量索引、全文索引、结构化索引，对应三路召回能力

#### 2. 多维度多路召回机制
- **向量检索**：采用代码专属 Embedding 模型，支持元数据前置过滤，适配模糊功能查找、语义相似匹配
- **关键词/全文检索**：针对代码标识符做分词优化，标识符加权匹配，适配精确符号定位
- **结构化检索**：基于知识图谱实现调用链追溯、依赖查询、继承关系筛选，适配关系类查询
- **融合重排层**：去重归一化后，按「语义相似度40% + 符号匹配30% + 结构关联20% + 热度10%」加权打分，输出 Top-N 结果

#### 3. 代码上下文理解完整链路
需求解析与召回策略选择 → 初筛召回定位核心代码节点 → 调用链上下层关联扩展（层级可控）→ Token 感知的分级裁剪 → 结构化注入 Agent 上下文

### （五）Agent 运行时引擎
- 驱动 ReAct、Reflexion 等经典 Agent 模式，管控「思考-决策-行动-观察」全循环
- 实时监控上下文 Token 水位，自动触发分级压缩策略
- 执行状态实时同步调度引擎与记忆引擎，保障全链路状态一致
- 支持单轮反思、多轮重试的执行控制，内置最大循环次数熔断

## 三、异常容错与质量保障体系
### （一）异常分级与结果校验
#### 1. 异常分级标准
- 瞬时异常：临时性、可自愈报错，采用指数退避重试
- 业务异常：参数、权限、逻辑类确定性失败，采用降级换路
- 质量异常：调用成功但结果不达标，采用重跑修正
- 致命异常：执行路径完全不可行，采用回滚+重规划/人工介入

#### 2. 三级结果校验机制
- **一级：规则化硬校验**（零成本，必执行）：格式校验、合规校验、边界校验、状态校验
- **二级：事实一致性校验**（工具类任务必执行）：比对输出结果与参考源（知识库、工具返回）的语义一致性
- **三级：目标完成度校验**（子任务/全局任务必执行）：调用轻量模型综合评分，按阈值分为通过、打回重跑、判定失败

### （二）分级容错机制
#### 1. 失败重试机制
- 准入规则：仅瞬时异常允许重试，确定性失败、合规问题禁止重试
- 分层重试策略：
  - 接口级：最多 3 次，指数退避，由模型/工具网关负责
  - 单步执行级：最多 2 次，附带修正提示，由 Agent 运行时负责
  - 子任务级：最多 2 次，重置上下文重跑，由任务调度引擎负责
- 配套保障：所有写操作幂等性保障、失败率过高自动熔断、重试时附带错误说明避免重复犯错

#### 2. 状态回滚与补偿机制
- 操作级回滚：写操作类工具配套注册 undo 补偿动作，执行失败自动触发
- 子任务级回滚：执行前打状态快照，失败逆序执行补偿，清理中间数据
- 全局任务级回滚：高风险写任务由全局事务协调器统一管理，全链路逆序回滚

#### 3. 降级与备用路径
- 资源级降级：模型、工具、Agent 主备资源池自动切换
- 功能级降级：非核心步骤可裁剪，输出中标注降级说明
- 管控规则：所有降级必须留痕标记，高风险零容错场景禁止自动降级

#### 4. 人工介入兜底
- 触发场景：自动机制全部失效、高风险任务终审不通过、用户主动申诉、系统级故障
- 操作能力：结果修正、路径调整、任务终止、问题标注
- 闭环优化：Badcase 回流系统，持续优化自动容错能力

### （三）配套保障机制
- 任务状态机严格管控所有状态流转，状态变更全量留痕
- 多层级成本熔断机制，避免异常场景成本失控
- 全链路 Trace 透传，所有异常可追溯定位

## 四、大模型幻觉治理体系
### （一）幻觉分类
1. 事实性幻觉：编造不存在的数据、事件、引用来源
2. 工具调用幻觉：选错/虚构工具、参数非法、虚构返回结果
3. 规划与逻辑幻觉：任务拆解遗漏、推导矛盾、执行路径不可行
4. 记忆幻觉：错误召回记忆、混淆会话、虚构历史交互
5. 格式幻觉：输出结构错乱、字段缺失、不符合约定规范

### （二）六层全链路治理架构
1. **第一层：模型选型与前置约束（源头防控）**
   - 分层模型路由，高风险环节采用低幻觉率强模型
   - 强约束系统提示词，强制思维链结构与来源标注要求
   - 事实类任务采用低温度参数，降低随机性

2. **第二层：推理过程自校验（生成中抑制）**
   - 分步推理+中间自检，每步校验通过后进入下一步
   - Reflexion 反思修正机制，多轮自我审查优化
   - 事实性内容强制标注来源，无来源判定为疑似幻觉
   - 信息不足时主动拒答，禁止强行编造

3. **第三层：外部知识与工具锚定（核心手段）**
   - RAG 全链路事实约束：强制召回知识库、信息不足拒答、内容绑定溯源ID
   - 工具调用硬管控：网关前置校验参数与权限、工具返回结果为唯一事实基准、核心数据多来源交叉验证
   - 结构化结果注入，明确区分事实数据与描述性内容

4. **第四层：多层级输出校验（后置兜底）**
   - 规则化硬校验：格式、来源、关键字校验
   - 事实一致性校验：语义比对输出与参考源的一致性
   - 综合质量终审：高风险场景启用强模型做全维度评估

5. **第五层：Agent 专属专项治理**
   - 记忆幻觉治理：写入校验、召回带源、定期蒸馏、分域隔离
   - 规划幻觉治理：DAG 合法性校验、执行偏差校验
   - 工具幻觉治理：网关前置拦截、结果语义对齐、执行结果强制覆盖

6. **第六层：长效闭环优化**
   - Badcase 全量归集与分类标注
   - 针对性迭代提示词、规则、知识库
   - 幻觉率指标持续追踪度量

### （三）分级治理策略
- 高风险场景（金融、法律、医疗、生产操作）：全六级治理 + 人工终审
- 中风险场景（调研、代码、数据分析）：前五级自动治理 + 抽样校验
- 低风险场景（文案、闲聊、信息整理）：前置约束 + 基础规则校验

## 五、静默漂移管控体系
### （一）漂移分类
1. 行为漂移：工具调用率、规划步数、输出风格长度、拒答率异常波动
2. 效果漂移：任务完成率、准确率缓慢下滑，幻觉率隐性上升
3. 对齐漂移：逐步偏离角色设定、合规边界、业务规则
4. 记忆漂移：错误/过时记忆累积，持续带偏推理
5. 模型漂移：底层大模型版本静默更新，整体能力分布偏移

### （二）四层管控闭环
1. **第一层：基准锚定体系**
   - 三类黄金基准集：行为基准、效果基准、对齐基准，作为漂移判定标尺
   - 全要素版本化管理：所有影响 Agent 行为的配置全部纳入版本管控，可追溯可回滚
   - 核心约束固化：系统提示词拆分核心约束区与业务配置区，压缩时优先保留核心规则

2. **第二层：多维度监测体系**
   - 行为指标监控：关注连续同向趋势变化，而非单次波动
   - 效果质量监控：在线抽样评测、定期全量回归、隐性用户反馈挖掘
   - 对齐合规监控：角色一致性检测、软违规内容占比趋势检测
   - 记忆质量监控：定期抽检错误记忆占比、召回相关性得分趋势

3. **第三层：分级自动纠偏**
   - 会话级轻度漂移：定期注入核心约束摘要，抵消上下文稀释
   - 任务级中度漂移：校验层打回纠正，切换更严格的 Prompt 模板
   - 系统级重度漂移：自动回滚至上一稳定版本，切换备用兜底模型
   - 记忆漂移专项：错误记忆标记失效、过期记忆自动归档、低相关记忆降权

4. **第四层：根因治理与长效闭环**
   - 标准化漂移归因，关联变更记录快速定位根因
   - 所有配置变更灰度发布，AB 对比无显著退化再全量
   - 定期全量记忆蒸馏校验，严格控制记忆写入门槛
   - 模型版本更新提前适配验证，保留稳定兜底版本

## 六、成本管控与模型选型体系
### （一）全链路分环节模型选型标准
| 链路环节 | 选型档位 | 核心诉求 |
|---|---|---|
| 意图识别/任务路由 | 轻量小模型 | 低成本、低延迟、高并发 |
| 任务规划/复杂推理 | 强模型 | 高准确率、低幻觉、强逻辑 |
| 工具调用/单步执行 | 中等模型 | 平衡效果与成本 |
| 结果汇总/格式校验 | 中等模型 | 低成本、格式合规 |
| 质量终审/高风险场景 | 强模型 | 零容错、效果兜底 |

### （二）核心成本测算规则
- 海外模型按输入、输出分开计费，输出单价通常为输入的 3~5 倍；中文 Token 密度为英文的 1.5~2 倍
- 单任务总成本 = 模型调用成本 + 工具调用成本 + 基础设施成本
- 代码等高消耗场景按简单/中等/复杂三个等级建立 Token 消耗基准

### （三）核心降本手段
分层路由降本、上下文压缩降本、缓存复用降本、记忆精准召回降本、流程优化降本、提示词精简降本
